HTML

  • HTML is short for HyperText Markup Language and is the standard language used for making websites.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The title element is used to specify the title of the webpage.
  • The header element represents a container for introductory content or a set of navigational links.
  • The section element represents a standalone section — which doesn't have a more specific semantic element to represent it — contained within an HTML document.
  • The footer element represents a footer for its nearest sectioning content or sectioning root element.
  • The a element is short for anchor and is used for hyperlinks - with the href
  • The img element is used to embed an image in an HTML page.
  • The ul element is used to represent an unordered list of items as a bulleted list.
  • The li element is used to represent an item in a list.

CSS

  • CSS stands for Cascading Style Sheets.
  • CSS describes how HTML elements are to be displayed on screen, paper, or in other media.
  • CSS saves a lot of work. It can control the layout of multiple web pages all at once.
  • the id selector refers to an element in the stylesheet by its id attribute.
  • the class selector refers to an element in the stylesheet by its class attribute.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A adds all files to the staging area ready to be committed
  • git commit -m "(your message here)" commits the changes to the local repository
  • git pull origin main: pulls the latest changes from the remote repository
  • git push origin main: pushes the latest changes to the main branch in the remote repository (github in our case)
